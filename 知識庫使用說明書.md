## 1. 知識庫架構一覽

知識庫分為**七個知識層**，前三層由 Cowork 透過 compile/ingest 生成，後四層由 Cowork 直接維護：

| 層 | 目錄 | 內容 | 生成方式 |
|----|------|------|---------|
| Layer 1 | `wiki/concepts/` | 核心概念（彎矩分配因子、傾角變位方程式等） | Cowork (compile-all) |
| Layer 1 | `wiki/problems/` | 題解析頁 | Cowork (ingest) |
| Layer 2 | `wiki/philosophy/` | 分析哲學：力法/位移法選擇邏輯 | Cowork (compile-all) |
| Layer 3 | `wiki/methods/` | 解題方法論（彎矩分配、傾角變位、矩陣法、最小功法） | Cowork (compile-all) |
| Layer 4 | `wiki/diagnosis/` | 題型診斷決策樹 | Cowork (直接存入) |
| Layer 5 | `wiki/failure-modes/` | 常見錯誤模式（符號錯誤/邊界條件/自由度/公式應用） | Cowork (直接存入) |
| Layer 6 | `wiki/materials/` | 材料特性補充（彈性模數、慣性矩計算） | Cowork (直接存入) |
| Layer 7 | `wiki/code-ref/` | 公式來源對應（結構學教材/考試常用公式） | Cowork (直接存入) |

**主導航入口：** `wiki/index.md`（七層全覽）、`wiki/by-year.md`（依考年）

## 2. 核心工作流程

```
【解析新考題】
  對 Cowork 說：「解析 XXXX 年考卷」
    ↓ Cowork 掃描 question_index.json，建立題目資料夾
  你把題目附圖截圖（fig-N.png）存入對應資料夾
    ↓
  告知 Cowork：「截圖完成，請開始解題」
    ↓ Cowork 一次解一題，存 .md 檔（含 LaTeX 公式）
  你驗算後對 Cowork 說：「將 SA-XXXX-N 的 verificationStatus 改為 verified」
    ↓
  對 Cowork 說：「ingest SA-XXXX-N」→ wiki 自動更新

【新增概念/方法】
  對 Cowork 說：「add concept [概念名]」或「add method [方法名]」
    ↓ 引導建立 concepts.json 條目 + wiki 頁面

【複習查詢】
  直接向 Cowork 提問，或瀏覽 wiki/ 目錄
```

> **所有操作都對 Cowork 說即可，不需要終端機。**

## 3. Cowork 指令速查

所有指令分三大類，完整說明見 [CLAUDE-CODE.md](CLAUDE-CODE.md)。

### 3.1 解題與 raw 資料維護

| 觸發語句 | 功能 |
|---------|------|
| `解析 XXXX 年考卷` | 掃描全年，建立題目資料夾並提醒截圖 |
| `繼續下一題` | 觸發解析同一份考卷的下一題 |
| `將 SA-XXXX-N 的 verificationStatus 改為 verified` | 人工驗算通過後標記 |
| `將 SA-XXXX-N 的 verificationStatus 改為 needs-review` | 發現錯誤時標記 |

### 3.2 wiki 編譯與維護（8個）

| 觸發語句 | 功能 | 前置條件 |
|---------|------|---------|
| `ingest SA-XXXX-N` | 將一道題目寫入 wiki | verificationStatus = "verified" |
| `compile all` | 從 raw/ 重建整個 wiki | — |
| `lint wiki` | 健檢 wiki 完整性（16 項） | — |
| `status` | 驗證進度統計 | — |
| `reindex` | 修正 hasSolution 不一致 | — |
| `add concept [名稱]` | 新增概念到 concepts.json | — |
| `add method [名稱]` | 新增解題方法論頁面 | — |
| `更新儀表板資料` | 重新生成 dashboard-data.js | — |

### 3.3 備考分析類（5個）

| 觸發語句 | 功能 | 輸出 |
|---------|------|------|
| `frequency` | 統計各考點歷年出現次數 | 對話回覆 |
| `analyze YYYY` | 深度分析某一年考卷的考點分布 | 對話回覆 |
| `predict` | 根據歷年頻率推測最可能出現的考點 | 對話回覆 |
| `study SA-UN` | 單元層級六區塊複習頁 | `study/study-SA-UN.html` |
| `study SA-UN-n` | 子項層級七區塊深度複習 | `study/study-SA-UN-n.html` |

### 3.4 查詢快捷類（4個）

| 觸發語句 | 功能 |
|---------|------|
| `find [關鍵字]` | 搜尋含某關鍵字的題目 |
| `related SA-XXXX-N` | 找出與某題考點相似的其他題目 |
| `unverified` | 列出所有有解析但尚未驗算的題目 |
| 直接提問 | 自由查詢知識庫 |

## 4. 複習使用指南

### 4.1 依考年複習

開啟 `wiki/by-year.md` → 選年份 → 點入題目頁 → 查看核心考點與解題步驟。

### 4.2 依題型複習（診斷決策樹）

| 題型 | 建議查詢 |
|------|--------|
| 靜定結構分析 | `wiki/diagnosis/statically-determinate.md` |
| 靜不定度判斷 | `wiki/diagnosis/indeterminacy.md` |
| 影響線 | `wiki/diagnosis/influence-line.md` |
| 彎矩分配法 | `wiki/diagnosis/moment-distribution.md` |
| 傾角變位法 | `wiki/diagnosis/slope-deflection.md` |
| 矩陣分析法 | `wiki/diagnosis/matrix-method.md` |
| 最小功法 | `wiki/diagnosis/minimum-energy.md` |

### 4.3 直接問 Cowork

```
彎矩分配法的分配因子怎麼算？
傾角變位法什麼時候要考慮側移？
SA-2018-3 的考點是什麼？
哪幾題考過影響線？
力法和位移法怎麼選？
```

## 5. 常見問題

**Q：解析 .md 放在哪裡？**
- 原始來源（唯讀）：`raw/solutions/SA-XXXX-N/SA-XXXX-N.md`
- 知識庫版本（ingest 後）：`wiki/problems/SA-XXXX-N.md`

**Q：如何更正已 verified 的答案？**
對 Cowork 說：「將 SA-XXXX-N 的 verificationStatus 改為 needs-review」→ 說明錯誤 → 重新驗算 → 改回 verified → 說「ingest SA-XXXX-N」。

**Q：截圖要怎麼命名？**

| 類型碼 | 內容 | 範例 |
|--------|------|------|
| `fig` | 題目附圖 | `SA-2018-1-fig-1.png` |
| `chart` | 彎矩圖/影響線截圖 | `SA-2018-1-chart-1.png` |
| `eqn` | 公式截圖 | `SA-2018-1-eqn-1.png` |
| `hand` | 手寫補充 | `SA-2018-1-hand-1.png` |

**Q：目前解析進度？**
對 Cowork 說 `status` 即可取得最新統計。
