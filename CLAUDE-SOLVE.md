**Cowork 執行前置（自動）：**
```

  - CLAUDE.md（身份層：分工、資料流、重要規則）

  - raw/json/question_index.json（確認哪些題目已有解析）

```

**執行規則：**
```
【階段一：建立資料夾並等待截圖（觸發語句後立即執行）】

1. 讀取 raw/exams/RC-YYYY_鋼筋混凝土設計與預力.pdf
2. 讀取 raw/json/question_index.json，找出尚無解析的題目

   → mkdir raw/solutions/RC-YYYY-N/（每題一個，已有解析者跳過）
4. 讀取考卷 PDF，瀏覽每道題目，判斷是否有附圖：
   → 有附圖的題目：列出，提醒使用者截圖並存入對應路徑：
       raw/solutions/RC-YYYY-N/RC-YYYY-N-fig-1.png
   → 無附圖的題目：標注「無需截圖」
5. 輸出截圖清單後停止，等待使用者通知「截圖完成，請開始解題」

7. 依序解析尚無解析的題目，但每次只解一題：
   → 解完一題，存檔後停止
   → 使用者確認後說「繼續下一題」，再解下一題
   → 目的：避免單次輸出超過 32,000 token 上限
8. 每道題目輸出一份獨立的 .md 檔案：
   → 直接存入 raw/solutions/RC-XXXX-N/RC-XXXX-N.md
9. 所有數學公式使用 LaTeX 格式
10. 每份解析開頭必須包含標籤區塊
11. 判斷是否需要輸出互動圖（依「圖形觸發條件」）：
```

#### 解題前置：三層掃描法（每題解題前先執行）

**核心原則：** 先見林（外在全貌）→ 再見樹（策略路徑）→ 後見土（內在意圖）

**第一層：外在掃描（2分鐘內完成）**
```
[目標] 這題最終要我交出什麼？（配筋量？驗算OK/NG？預力損失？P-M互制？）
[已知] 題目給了哪些數字和條件？（斷面、f'c、fy、載重、幾何、配筋、參考公式）
[待算] 從已知到目標，需要算出哪些中間參數？（Mn, Vc, Ie, Pe, 損失...）
```

**第二層：策略掃描**
```
[邏輯] 適用哪個設計邏輯？（見下方「設計哲學框架」）
[陷阱] 考官可能在哪裡挖了坑？（平衡鋼筋比邊界、有效深度 d、T形梁翼板寬、損失順序）
[捷徑] 有沒有條件可以跳過某些步驟？（無腹筋梁→只驗Vc、ρ≪ρb→單筋梁）
```

**第三層：內在掃描**
```
[核心] 這題最重要的一個觀念是什麼？（強度折減哲學、延性控制、有效預力）
[線索] 特殊用字給了什麼暗示？（「韌性設計」→耐震配筋加嚴、「服務載重」→撓度/裂縫）
```

#### 設計哲學框架（解題邏輯對照）

**RC 梁彎曲設計（USD）**
```
核心：φMn ≥ Mu（強度需求 ≤ 設計強度）
步驟：① 判斷梁型（單筋/雙筋/T形梁）
      ② Whitney 等值矩形應力塊：a = As·fy / (0.85·f'c·b)
      ③ 標稱彎矩：Mn = As·fy·(d - a/2)
      ④ 延性檢核：ρmin ≤ ρ ≤ ρmax（或 εt ≥ 0.004）
      ⑤ φ 值：拉力控制 εt ≥ 0.005 → φ=0.9；過渡區線性插值
關鍵：平衡鋼筋比 ρb = 0.85·β1·f'c/fy · 6120/(6120+fy)（kgf/cm² 單位）
      最大鋼筋比 ρmax ≈ 0.75ρb 或依延性條件控制
```

**T 形梁設計**
```
核心：判斷中性軸位置（翼板內 or 腹板中）
步驟：① 有效翼板寬（取小值）：
         - 內梁：L/4、bw + 8hf×2、梁中心距
         - 邊梁：L/12 + bw、bw + 6hf、邊距
      ② 假設 a ≤ hf → 計算 a，驗證假設
      ③ 若 a > hf → 分離 Asf（翼板壓力部分）與 Asw（腹板部分）
      ④ Mn = Mnw + Mnf
```

**RC 梁剪力設計（USD）**
```
核心：φVn ≥ Vu；Vn = Vc + Vs
步驟：① 臨界斷面：距支承面 d 處
      ② Vc = (1/6)·√f'c·bw·d（N, mm；SI 單位）
         或 Vc ≈ 0.53√f'c·bw·d（kgf/cm²；台灣慣用）
      ③ 若 Vu > φVc → 需配腹筋：Vs = Av·fy·d/s
      ④ 最大間距：s ≤ d/2 且 s ≤ 600mm（Vs ≤ (1/3)√f'c·bw·d 時）
                   s ≤ d/4 且 s ≤ 300mm（Vs > (1/3)√f'c·bw·d 時）
      ⑤ 最小腹筋：Av,min = 0.35·bw·s/fyt（kgf/cm²）
```

**RC 柱設計（USD）**
```
核心：φPn ≥ Pu；φMn ≥ Mu（P-M 互制圖）
純軸壓上限：φPn,max = 0.80φ[0.85f'c(Ag-Ast) + fyAst]（矩形箍筋柱）
             φPn,max = 0.85φ[0.85f'c(Ag-Ast) + fyAst]（螺旋柱）
φ 值：純壓力控制 φ=0.65（矩形箍）/ 0.75（螺旋）；拉力控制 φ=0.9
配筋限制：0.01 ≤ ρg ≤ 0.08
```

**P-M 互制圖關鍵點**
```
① 純軸壓點（e=0）：Pn,max
② 平衡點（εt = εy）：
   cb = 6120/(6120+fy) · d（kgf/cm²）
   ab = β1·cb
   Pb = 0.85f'c·ab·b + Cs - T
   Mb = Pb·eb
③ 純彎點（P=0）：Mn（同梁計算）
④ 中間任意點：假設 c → 應力分布 → Pn, Mn, e = Mn/Pn
```

**長柱效應（放大彎矩法）**
```
無側移構架（δns）：Mc = δns·M2
  δns = Cm/(1 - Pu/0.75Pc) ≥ 1.0
  Pc = π²EI/(klu)²；EI = 0.4EcIg/(1+βd)
  Cm = 0.6 + 0.4(M1/M2) ≥ 0.4（端部有側向載重時 Cm=1.0）
有側移構架（δs）：Mc = δs·M2
  δs = 1/(1 - ΣPu/0.75ΣPc)
判斷側移：klu/r ≤ 34-12(M1/M2)（無側移）；klu/r ≤ 22（有側移可忽略）
```

**撓度計算（ACI 有效慣性矩法）**
```
Ie = (Mcr/Ma)³·Ig + [1-(Mcr/Ma)³]·Icr ≤ Ig
Mcr = fr·Ig/yt；fr = 2√f'c（kgf/cm²）或 0.7√f'c（MPa）
Icr：開裂斷面慣性矩（需計算中性軸位置）
長期撓度修正：Δtotal = Δi(DL)·(1+λ) + Δi(LL)
  λ = ξ/(1 + 50ρ')；ξ=2.0(≥5年) / 1.4(1年) / 1.2(6月) / 1.0(3月)
```

**預力混凝土設計**
```
核心：有效預力 Pe = Pi × (1 - 損失率)
損失分類：
  【立即損失（施工時）】
  ① 彈性縮短：ΔfpES = (Ep/Ec)·fcpa
     先拉法 × n（一次全部）；後拉法 × (n-1)/2n·n（分批近似）
  ② 摩擦損失（後拉）：ΔfpF = fpi·[1 - e^-(μα + Kx)]
     μ = 摩擦係數；α = 角度改變；K = 非預期彎曲係數
  ③ 錨定滑動損失（後拉）：ΔfpA
  【時間依存損失（長期）】
  ④ 潛變：ΔfpCR = Cr·(Ep/Ec)·(fcpa - fcpq)
  ⑤ 收縮：ΔfpSH = Ksh·Ep·ε_sh（估算值依規範查表）
  ⑥ 鬆弛：ΔfpR（依鋼絞線等級：低鬆弛 vs 一般鬆弛）
使用性驗算（彈性理論）：
  f = Pe/A ± Pe·e·y/I ± M·y/I
  壓應力 ≤ 0.45f'c（ACI）；拉應力限制視狀況
```

**耐震設計（特殊韌性構架）**
```
核心：強柱弱梁、塑鉸位置於梁端、能量消散
梁端配筋限制：
  頂底筋量 ≥ 2 根且 ρ ≥ ρmin
  底筋正彎矩強度 ≥ 負彎矩強度 × 1/2（各端）
  全長正/負彎矩強度 ≥ 柱面最大彎矩 × 1/4
  拉壓筋比 ρ'/ρ ≥ 1/4（慣用要求）
柱圍束箍密區：
  長度 lo = max(淨高/6, 最大邊長, 450mm)
  間距 so ≤ min(最小邊長/4, 6db, 150mm)
  Ash ≥ max(0.3·s·hc·f'c/fyh·(Ag/Ach-1), 0.09·s·hc·f'c/fyh)
強柱弱梁：ΣMnc ≥ (6/5)·ΣMnb
```

```
   - 推測出題者測驗的能力、想避開的陷阱

## 3. 解題戰略地圖與陷阱分析 (Strategic Roadmap & Trap Analysis)
   - 步驟化作戰計畫（先算什麼、後算什麼）
   - 明確列出 3-4 個關鍵陷阱及應對策略

## 3.5 變數層次分析 (Variable Hierarchy Analysis)  ← 必須輸出
   > 複習提示：第一次解題後，在每個卡住的知識點旁標記 `⚠`；第二次複習時只看有 `⚠` 的項目。

   ### 最終目標
   `一句話說明本題最終要求出什麼`

   ### 本題關鍵公式（依計算順序）
   - 列出解題主幹公式（5–8 個 Step），**不含數字代入**，只顯示符號結構
   - 規則：**已由前步驟推導出的中間變數**（非 L1 直接給定），在後續公式中出現時用 `\boxed{}` 框選
     - 示例：Step 1 求出 $k_y$；Step 2 用到它時寫 $\phi_y = \varepsilon_y / [d(1-\boxed{k_y})]$
     - 目的：讓考生一眼看出哪些量是「需要先算出來才能帶入」的
   - 格式：每個 Step 獨立一行 display math，加簡短文字說明

   ### L1：題目直接給定
   _看到題目就能讀出的數字，不需要任何公式。_
   - 欄位：**符號 ∣ 數值 ∣ 說明**（無卡關欄）

   ### L2：需知識點推導
   _需要知道公式名稱與適用條件，套入 L1 即可算出。_
   - 按計算步驟分組，每組加粗標題
   - 欄位：**符號 ∣ 公式／來源 ∣ 卡關?**（卡關欄全部留空，由考生複習時自填 ⚠）

   ### L3：深層知識（不懂就卡住）
   - 從陷阱分析 §3 提取，列出考生若觀念不清就會在 L2 用錯公式的知識點
   - 欄位：**知識點 ∣ 說明 ∣ 卡關?**（卡關欄全部留空）
```
